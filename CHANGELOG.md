# mayatk — Changelog

## 2026

- **New tool — Curtain (`edit_utils/curtain.py`).** Procedural draped-cloth generator, split by responsibility (SRP): **`Rail`** (rail geometry — generate / resolve-from-selection / sample / measure / resample), **`CurtainMesh`** (the *deformation* — drape a grid into pleated, gravity-sagged cloth), and **`CurtainRig`** (the *rig* — `_add_wire` deformer + `_add_clusters` controls, orchestrated by `attach`). The cloth hangs from a *rail* — a selected NURBS curve, polygon edge path, locator chain, or a generated **straight** rail (bow it with `curvature`; the default is flat) — and is pinned at N evenly-spaced **hanging points**. Each hanging point is a pleat (the fabric gathers there); the spans between them belly into folds and sag down a true **catenary** (`y = a·cosh(x/a)`, `catenary_shape()`) — wider gaps fall further, scaled by `gravity`. Controls: `height`, `hanging_points`, `gravity`, `tension` (catenary shape; peak stays at 1 so `gravity` owns the depth), **`round_points`** (round the sharp cusp at each pin into a smooth dome — `sag_profile()` blends the catenary toward `sin²`), `fullness`, `taper` (top-gather ↔ hem-flare), **`mid_folds`** + **`fold_seed`** (seeded **V-shaped creases** that radiate down from random points near the top and run various lengths — the diagonal break-lines of gathered fabric), **`end_bend_left`/`end_bend_right`/`end_bend_falloff`** (signed per-end sideways bend with an edge ramp — e.g. panels curling toward the camera), `irregularity` (now dialled much subtler — the deliberate relief comes from fullness/mid_folds), `seed`, `density` (segments/unit), `reduce` (polyReduce decimation %), `thickness`, **`invert`** (reverse normals), `closed`. Mesh built once via `polyPlane(ch=False)` + `MFnMesh.setPoints`; vector math delegates to `ptk.MathUtils`. `CurtainSlots` drives it through the hermetic `Preview` (live re-drape via `connect_multi`); a rail is **auto-created on preview** when none is selected (so you never have to select an unrelated object) and dropped on commit. In generated mode the drape reads **Width/Curvature live each refresh** (previously the captured auto-rail froze them, so they appeared to do nothing); the owned driver curve resyncs only when a rail-shaping field actually changes. The **preset combo lives in the panel** (`cmb000`); ships built-in presets (Flat Backdrop, Stage Swag, Shower Curtain, and a closed-ring Booth Ring) through uitk's `PresetManager` (built-in tier from `edit_utils/presets/curtain/`). The panel groups its fields into collapsible `uitk.CollapsableGroup` sections (Rail, Hanging, Drape, Folds/Noise, Ends, Mesh — the niche Folds/Noise and Ends start collapsed, each section's open/closed state persisted across sessions) with spinbox value columns aligned per group via font metrics (`_align_spinbox_prefixes`). Mesh-softening is always on (no toggle). `CurtainRig` (wire deformer + per-CV cluster controls) remains a tested library primitive for programmatic rigging but is no longer wired into the panel. Registered `edit_utils.curtain: (CurtainMesh, Rail, CurtainRig)`; launched from tentacle's Deformation panel. Covered by `test/test_curtain.py` (35 tests).
- **Cut On Axis preview — fixed cuts stacking on historyless meshes.** Changing a value (Cuts/Offset/Axis/Pivot) while Preview was on added a *new* cut without rolling back the previous one — on frozen/imported meshes only. Root cause: `polyCut(ch=True)` on a mesh with no upstream history spawns an intermediate orig-shape that holds the only pristine copy; the hermetic preview's node-diff rollback deleted that orig-shape (counted as a "created" node), which *baked* the cut into the visible mesh instead of reverting it. Fix is in the shared preview contract: a new `PRESERVE_GEOMETRY = True` opt-in (set on `CutOnAxisSlots`) makes `CleanupContract` snapshot the operated meshes and, on rollback, restore in place any that survived but diverged from the snapshot — preserving the original shape's identity/UUID and skipping the restore when node-diff already reverted (so meshes *with* legitimate history keep it). Regression coverage in `TestCutOnAxisPreviewRollback` (`test_edit_tools_geometry.py`) drives the real `Preview` through enable→refresh→disable on a historyless cube. Mirror / duplicate (`MUTATES_SELECTION`) paths unchanged.

Import the live-session RPC client from `mayatk.mat_utils.marmoset_bridge.marmoset_rpc` (was `pythontk.marmoset_utils.marmoset_rpc`). Public `MarmosetBridge` API unchanged. The `test_toolbag_helpers` / `test_marmoset_rpc` suites moved into `test/mock_tests/`; mock + live bridge tests updated.
- **Marmoset bridge split — engine moved to `pythontk.marmoset_utils`** (breaking, no shim). `MarmosetBridge` is now a thin `pythontk.marmoset_utils.MarmosetEngine` subclass that keeps only the Maya glue — FBX export of the selection, the `MatManifest` material sidecar, and DAG-based high/low bake-pairs classification — and overrides `send(objects=...)` to delegate to the engine with the produced file paths. The Toolbag-side helpers, `templates/`, `marmoset_rpc`, and the log/launch/render logic all moved upstream. Import the live-session RPC client from `pythontk.marmoset_utils.marmoset_rpc` (was `mayatk.mat_utils.marmoset_bridge.marmoset_rpc`). `parameters.py` (AttributeSpec widget specs) + the Switchboard slots/`.ui` stay here; the slots' log-helper imports now come from `pythontk.marmoset_utils.toolbag_log`. Mock + live bridge tests updated (37 mock tests pass).

Same same-name+same-size collision protection as Normalize Paths — different-size collisions skip-and-warn rather than silently rebinding to a wrong file.
  - **Find & Copy Textures** (`tb_find_and_copy_textures`) gains a Copy/Move combobox; the button text auto-syncs to match the active mode (*Find & Copy Textures…* ↔ *Find & Move Textures…*) so the active verb is visible on the menu item itself. Move uses `MatUtils.move_texture_files(delete_old=True)`.
  - **Reload Scene Textures** — new plain QPushButton under General that calls `MatUtils.reload_textures(refresh_viewport=True)` so Maya re-reads every file/aiImage/pxrTexture/imagePlane node from disk. Useful after Find & Copy / Normalize Paths relocate files.
  - Tooltip cleanup: removed "(▸)" option-box hints (it's a known UI element); Resolve Missing tooltip now names the cascade order explicitly (`Stem → Texture → Fuzzy`); Normalize combobox labels say "external textures" instead of bare "externals".
- **Tool help text migrated from header-menu "Instructions" buttons to the new `Header.set_help_text(...)` API** — every Switchboard tool used to bury its instruction tooltip behind an `Instructions` `QPushButton` in `widget.menu` (under a `Separator → "About"` divider). The new uitk `Header` exposes the help text via a leftmost `?` button in the header bar; hovering shows the tooltip and clicking pops it open. 38 mayatk slot classes migrated (audio_clips, blendshape_animator, shots, shot_manifest, shot_sequencer, color_manager, exploded_view, bevel, bridge, cut_on_axis, duplicate_{grid,linear,radial}, dynamic_pipe, mirror, naming, snap, hierarchy_manager, reference_manager, scene_exporter, workspace_map, hdr_manager, game_shader, image_to_plane, marmoset_bridge, mat_updater, render_opacity, shader_templates, substance_bridge, texture_path_editor, channels, image_tracer, shadow_rig, telescope_rig, tube_rig, wheel_rig, calculator, rizom_bridge). See [`m3trik/docs/HEADER_HELP_INVENTORY.md`](../m3trik/docs/HEADER_HELP_INVENTORY.md) for the full migration list and the uitk-side implementation notes.

- **Texture Path Editor consolidation** — collapsed the path-management menu surface from 13 entries to 5 (4 header + 1 context). Mirrored global/row commands replaced with smart scope (selection if any, else all). The six "Resolve Missing by …" variants fold into one **Resolve Missing Textures** action button with strategy checkboxes (Stem / Texture / Fuzzy; cascade in safest-first order, stop at first hit) in its option_box flyout. **Set Directory** / **Find & Copy Textures** / **Normalize Paths** replace `lbl010` / `lbl_find_copy` / `lbl013` plus their per-row twins. **Normalize Paths** is new behavior: rewrite paths under sourceimages to relative without moving files, UDIM-safe, with an option_box combobox controlling externally-located valid textures — *Leave untouched* (default), *Copy to sourceimages*, or *Move to sourceimages*. Collision policy: same-basename + same-size in sourceimages = safe rebind without overwriting; different size = skip + warn (never silently rebind to a wrong texture, never destroy the external). The two action items that actually carry options — **Normalize Paths** (`tb_normalize_paths`) and **Resolve Missing Textures** (`tb_resolve_missing_textures`) — are uitk `PushButton`s (`tb_*` per convention) auto-wired by name, with per-button option_box flyouts populated by matching `_init` methods (standard pattern, cf. `naming_slots.tb003_init`, `audio_clips_slots.tb001_init`). Plain action items stay as `QPushButton` wired via `clicked.connect`. No inline menu state widgets, no modal dialogs. Added **Select Absolute Paths** as a counterpart to Select Broken Paths for previewing what Normalize would touch. Removed slots: `lbl010`, `lbl_find_copy`, `lbl013`, `lbl014`, `lbl015`, `resolve_missing_by_{stem,fuzzy,texture}`, `row_set_texture_directory`, `row_find_and_copy_texture`, `remap_to_relative`, `row_resolve_by_{stem,fuzzy,texture}` — replaced by `set_texture_directory`, `find_and_copy_textures`, `tb_normalize_paths`, `tb_resolve_missing_textures`, `select_textures_for_objects`, `select_broken_paths`, `select_absolute_paths`, `open_source_images`. Any custom hotkeys or scripts referencing the old slot names will need rebinding.
- **Channels wheel-scroll — symmetric ladder + axis-swap fix** — wheel-scroll on a channel-box value cell now mirrors uitk `SpinBox`'s symmetric ladder. The two user-visible bugs both root in the table layer:
  - **Alt-held wheels did nothing**: `TableWidget._emit_wheel_scrub` read only `event.angleDelta().y()`, which is 0 on some Qt builds when Alt is held (Qt transposes the delta onto `.x()`). The widget now reads `delta.y() or delta.x()`, so Alt and Ctrl+Alt both fire reliably. (Fix in uitk; the channels module is just the consumer.)
  - **Ctrl+Shift fell to the default 0.1 step**: the old `_wheel_step` matched modifiers with `(mods & MASK) == MASK`, which can return False under PySide6 when the live `KeyboardModifiers` flag set is compared to a `KeyboardModifier` flag value. Detection now uses `bool(mods & SPECIFIC)` per modifier, making the comparison robust to PySide5/6 type quirks.
  - **Modifiers source**: `_on_wheel_scrolled` now takes the modifiers from the `cellWheelScrolled` signal payload (added in uitk) rather than polling `QApplication.keyboardModifiers()`. Removes a race against the wheel event.
  - **Ladder** (matches uitk `WheelStepMixin`):

        plain         ×1       (default)
        Ctrl          ×10      (coarse)
        Ctrl+Shift    ×100     (very coarse)
        Alt           ÷10      (fine)
        Ctrl+Alt      smallest representable step (10⁻ᵈᵉᶜⁱᵐᵃˡˢ; for ints = 1)

    For int attrs `Alt` collapses to 0 (no sub-1 step exists) -- the wheel notch is honestly a no-op there. The earlier *Ctrl+Alt = snap-to-integer* iteration was reverted: it visually matched `Ctrl` for any integer-aligned starting value, so it looked broken even when working correctly. The smallest-step gesture is now distinct from `Ctrl` regardless of value alignment.
  - **Breaking**: `WHEEL_MOD_FINE` / `WHEEL_MOD_SMALLEST` class attributes (overrideable in subclasses) are gone; per-app hotkey customization will need a different mechanism if/when it's needed.

- **RizomUV Bridge unwrap fix + hard/organic split** — replaced the broken `unwrap.lua` (passed `Auto={HardEdge=true, ..., Skeleton={}}`, which RizomUV silently ignores → zero seams selected → mesh re-imported as one welded shell, i.e. "all edges sewn"). Two new presets ship: `unwrap_hard.lua` (driven by `Auto={SharpEdges={AngleMin=…}, QuadLoopCutter=true, ...}`, canonical Titus/3ds-max signature) and `unwrap_organic.lua` (QuadLoopCutter off, higher default angle, leans on HandleCutter / PipesCutter / StretchLimiter). New `SHARP_ANGLE` parameter exposes the dihedral threshold in the panel.
- **RizomUV Bridge pack fix** — wrapper `ZomLoad` was `XYZ=true` (positions only), so incoming UVs were dropped on load. Pack/optimize then ran on zero islands and the file round-tripped with degenerate UVs ("UVs gone"). Wrapper now loads `XYZUVW=true, UVWProps=true` to bring mesh + existing UVs through together. Matches the C4D bridge and Titus batch-load signatures.
- **RizomUV Bridge pack attributes exposed** — added five user-tunable `ZomPack` knobs: `PACK_RESOLUTION` (texel-density baseline), `PACK_MAX_MUTATIONS` (solver iterations), `PACK_ROTATE_ENABLE`, `PACK_TRANSLATE`, `SCALING_MIX`. Hardcoded `Translate=true` and the absent `Rotate.Enable`/`MaxMutations`/`Resolution` are now driven from the panel; preset Lua across `pack`/`optimize`/`unwrap_hard`/`unwrap_organic` updated in lockstep. `Margin` and `Quality` remain absent (still crash 2020.1, comment in `parameters.py`).
- **RizomUV Bridge FBX UV-set + normalize fix** — wrapper was passing `FBX=true` (not a real Rizom flag — silently dropped, UV-set name not preserved) and omitting `NormalizeUVW`, which defaults to `true` and rescales the incoming layout to 0-1 (corrupts pack/optimize input). Wrapper now emits the canonical `FBX={UseUVSetNames=true}` nested table (matching the adevra Maya-bridge and Titus commented reference) and explicit `NormalizeUVW=false` on load. Fixes "UVs come back gone" on optimize/pack round-trips where the round-trip writes UVs into a different UV-set name than the bridge then reads back.
- **RizomUV Bridge optimize-preset rewrite + Resolution combobox** — `optimize.lua` now uses the Titus optimize signature: `ZomIslandGroups({Mode="DistributeTilesContent", FreezeIslands=true, UseTileLocks=true, UseIslandLocks=true, GroupPaths={"RootGroup"}})` followed by a `Scaling={Mode=0, Mix=true}` + `LayoutScalingMode=0` `ZomPack`. Layout-preserve invariants are hardcoded so the pre-scale / layout-scale / mix-scale widgets auto-hide for this preset — relax/rotate/recursion/resolution/mutations stay user-tunable. `PACK_RESOLUTION` is now a `choice` combobox (256/512/1024/2048/4096/8192) instead of a free-form int spinbox.
- **RizomUV Bridge version-gated params** — `MaxMutations` / `Resolution` / `Rotate.Enable` and the FBX-load `FBX={UseUVSetNames=true}` table all crash RizomUV 2020.1 with an access violation (same failure mode as the already-skipped `Margin` / `Quality`). Bridge now parses the version out of the install path (`RizomUV 2020.1` → `(2020, 1)`) and uses `MIN_VERSIONS` in `parameters.py` to gate them: the panel hides the affected widgets and `_construct_full_script` strips the corresponding lines before send. Gate threshold is `(2022, 0)` — adjust downward if an earlier release is confirmed to support these. Slot wires the strip in via a `_VersionedParamsProxy` so `referenced_keys` is version-aware everywhere it's called.
- **RizomUV Bridge optimize-preset second pass + better errors** — even with version-gated params, `optimize.lua` was still crashing 2020.1; the remaining culprits were the Titus-style `ZomIslandGroups({Mode="DistributeTilesContent", FreezeIslands=true, UseTileLocks=true, UseIslandLocks=true, GroupPaths={...}})` block plus the `AuxGroup` / `WorkingSet` keys on `ZomPack`. Preset now mirrors `pack.lua`'s `DistributeInTilesEvenly` / `MergingPolicy=8322` / `GroupPath` (singular) pattern but keeps the layout-preserve intent via scalar-only `Scaling={Mode=0, Mix=true}` + `LayoutScalingMode=0`. Tile-assignment preservation is lost (was Titus-only) but the relax+repack workflow works on 2020.1. Wrapper also drops the speculative `NormalizeUVW=false` (Titus Batch reference omits it with XYZUVW). Crash error message now includes the tail of Rizom's stdout/stderr and the detected version — the bare `exit code N` was useless for debugging.
- **ReferenceManager filter toggle migration** — `txt001_init` now uses `uitk.ToggleOption` (via `option_box.set_toggle`) instead of a hand-rolled two-state `ActionOption`. Eliminates the local enable/disable cycle wiring; the toggle owns its visuals (red when disabled, per `pythontk.Palette.status()["error"]`) and emits a single `toggled(bool)` signal connected to `_toggle_filter`.
  - **Persistence fix**: filter state now survives across Maya sessions correctly. The previous `set_action(states=…)` form auto-persisted `current_state` keyed off the widget's `objectName`, but `self.controller._filter_enabled` reset to `True` on every session start — producing a visual/flag desync where the icon could read "disabled" while the controller had reverted to enabled, requiring two clicks to actually disable. The new wiring gives the toggle a dedicated persistence key (`"reference_manager_filter"`), then syncs `self.controller._filter_enabled` from the toggle's restored state at `txt001_init` so the icon and predicate flag always agree.
