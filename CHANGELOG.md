# mayatk ‚Äî Changelog

## 2026

- **`EditUtils.dissolve_coplanar()` ‚Äî planar decimation (limited dissolve).** Angle-driven, feature-preserving complement to `decimate`: walks each mesh's interior edges (OpenMaya) and `polyDelEdge(cleanVertices=True)`-merges those whose two faces are within `angle_tolerance` degrees of coplanar, leaving creases/corners/silhouette/borders untouched and keeping clean n-gons (no triangulation). At a tiny tolerance it's **lossless** on hard-surface meshes (a 5√ó5√ó5 subdivided cube ‚Üí 6 faces, bounding box unchanged); curved surfaces are left alone (use `decimate`/QEM there). Tentacle's Subdivision **Decimate** button (now `tb000`) exposes both algorithms behind an option-box combobox ‚Äî Reduce (QEM %) vs Planar (Coplanar) ‚Äî with the irrelevant options greyed out per selection. Tests: `test_edit_utils.py` (cube‚Üí6 lossless, sphere untouched).
- **`EditUtils.decimate()` ‚Äî reusable polyReduce helper; Curtain math promoted to pythontk.** Extracted the scattered `polyReduce` logic (curtain's mesh reduce + the Subdivision panel's Reduce button) into one boundary-preserving **`EditUtils.decimate(objects, percentage, preserve_borders, preserve_hard_edges, preserve_uv_borders, preserve_quads, symmetry, ‚Ä¶)`** (undoable, deletes history by default; early-returns at `percentage <= 0`); the Curtain generator's `reduce` now routes through it, and tentacle's new Subdivision **Decimate** button calls it. Separately, the Curtain module's general math/signal primitives were promoted to `pythontk.MathUtils` (the SSoT): `smoothstep`, `ricker`, `catenary`/`catenary_sag`, `safe_normalize`, vector `lerp`, and the coherent grain generator (now `pythontk.BandLimitedNoise`). `curtain.py` consumes those via thin aliases (`catenary_shape`/`sag_profile` remain importable for back-compat) and keeps only the curtain-specific `_v_arms` fold composition local. No behavior change ‚Äî the noise is bit-identical (same seed/octave draw order); `test_curtain.py` (43) + `test_edit_utils.py` (+2 decimate tests) stay green.
- **Curtain ‚Äî real "Mid Folds" V-forks, renamed creases, dropped dead seed/presets.** The old **Mid Folds** slider only added V-shaped creases ‚Äî it never *folded* anything ‚Äî so it's renamed **Creases** (`creases` + `crease_seed`, was `mid_folds`/`fold_seed`; behavior unchanged). **Mid Folds** is now a genuine adjustment matching how a heavy gathered drape actually forks (`mid_folds` intensity + `mid_fold_seed`): on a seeded random ~1/4‚Äì1/2 of the *interior* hang points, a downward **V** apexes *on* the hang point and fans its two arms out and down into the neighbouring spans, interrupting their plain in/out belly. Lengths vary (some stop high, some run nearly to the hem) as do width/depth (narrow-deep vs wide-shallow); the seed does most of the look's work (`_make_midfolds`/`_midfold_offset`). Modelled in global `u` (continuous across hang points by construction ‚Äî no per-span cracks) on a shared `_v_arms()` primitive that also backs `_crease_offset`. Each fold arm is a **Ricker/Mexican-hat wavelet** (`_ricker()`, the zero-integral negated 2nd-derivative-of-gaussian used for wrinkle synthesis), so a crease pushes the cloth *out* at its line and *in* to either side ‚Äî material-conserving the way fabric buckles, rather than ballooning one way (fixes folds reading only as outward bulges); arm spread + fold width are span-relative so the fork shape is invariant to hang-point count, while depth stays absolute. The dead **Seed** field (`seed`) is gone ‚Äî it was never wired into the engine and drove only the always-subtle grain/jitter, which now uses a fixed internal seed (`_BASE_SEED`). **`irregularity`** is now **coherent, band-limited** surface grain ‚Äî a few smooth zero-mean wave octaves (`_make_billow`/`_billow_offset`, 1/f amplitude roll-off, horizontal frequency leading vertical so it reads as soft vertical-ish wrinkles) ‚Äî replacing the old per-vertex white noise that read as static; the per-vertex RNG is dropped from `_drape`. UI: `grp_folds` reordered to Mid Folds (`s019`, new) / Mid Fold Seed (`s010`, repurposed from the dead Seed) / Creases (`s014`) / Crease Seed (`s015`) / Irregularity (`s008`). Dropped the **Flat Backdrop** and **Booth Ring** built-in presets (Stage Swag + Shower Curtain remain, with their creases preserved). Tests renamed (`test_creases_*`) and extended (`test_midfolds_*`, incl. direct checks that forks anchor on a hang-point subset and leave the top edge pinned).
- **New tool ‚Äî Curtain (`edit_utils/curtain.py`).** Procedural draped-cloth generator, split by responsibility (SRP): **`Rail`** (rail geometry ‚Äî generate / resolve-from-selection / sample / measure / resample), **`CurtainMesh`** (the *deformation* ‚Äî drape a grid into pleated, gravity-sagged cloth), and **`CurtainRig`** (the *rig* ‚Äî `_add_wire` deformer + `_add_clusters` controls, orchestrated by `attach`). The cloth hangs from a *rail* ‚Äî a selected NURBS curve, polygon edge path, locator chain, or a generated **straight** rail (bow it with `curvature`; the default is flat) ‚Äî and is pinned at N evenly-spaced **hanging points**. Each hanging point is a pleat (the fabric gathers there); the spans between them belly into folds and sag down a true **catenary** (`y = a¬∑cosh(x/a)`, `catenary_shape()`) ‚Äî wider gaps fall further, scaled by `gravity`. Controls: `height`, `hanging_points`, `gravity`, `tension` (catenary shape; peak stays at 1 so `gravity` owns the depth), **`round_points`** (round the sharp cusp at each pin into a smooth dome ‚Äî `sag_profile()` blends the catenary toward `sin¬≤`), `fullness`, `taper` (top-gather ‚Üî hem-flare), **`mid_folds`** + **`fold_seed`** (seeded **V-shaped creases** that radiate down from random points near the top and run various lengths ‚Äî the diagonal break-lines of gathered fabric), **`end_bend_left`/`end_bend_right`/`end_bend_falloff`** (signed per-end sideways bend with an edge ramp ‚Äî e.g. panels curling toward the camera), `irregularity` (now dialled much subtler ‚Äî the deliberate relief comes from fullness/mid_folds), `seed`, `density` (segments/unit), `reduce` (polyReduce decimation %), `thickness`, **`invert`** (reverse normals), `closed`. Mesh built once via `polyPlane(ch=False)` + `MFnMesh.setPoints`; vector math delegates to `ptk.MathUtils`. `CurtainSlots` drives it through the hermetic `Preview` (live re-drape via `connect_multi`); a rail is **auto-created on preview** when none is selected (so you never have to select an unrelated object) and dropped on commit. In generated mode the drape reads **Width/Curvature live each refresh** (previously the captured auto-rail froze them, so they appeared to do nothing); the owned driver curve resyncs only when a rail-shaping field actually changes. The **preset combo lives in the panel** (`cmb000`); ships built-in presets (Flat Backdrop, Stage Swag, Shower Curtain, and a closed-ring Booth Ring) through uitk's `PresetManager` (built-in tier from `edit_utils/presets/curtain/`). The panel groups its fields into collapsible `uitk.CollapsableGroup` sections (Rail, Hanging, Drape, Folds/Noise, Ends, Mesh ‚Äî the niche Folds/Noise and Ends start collapsed, each section's open/closed state persisted across sessions) with spinbox value columns aligned per group via font metrics (`_align_spinbox_prefixes`). Mesh-softening is always on (no toggle). `CurtainRig` (wire deformer + per-CV cluster controls) remains a tested library primitive for programmatic rigging but is no longer wired into the panel. Registered `edit_utils.curtain: (CurtainMesh, Rail, CurtainRig)`; launched from tentacle's Deformation panel. Covered by `test/test_curtain.py` (35 tests).
- **Cut On Axis preview ‚Äî fixed cuts stacking on historyless meshes.** Changing a value (Cuts/Offset/Axis/Pivot) while Preview was on added a *new* cut without rolling back the previous one ‚Äî on frozen/imported meshes only. Root cause: `polyCut(ch=True)` on a mesh with no upstream history spawns an intermediate orig-shape that holds the only pristine copy; the hermetic preview's node-diff rollback deleted that orig-shape (counted as a "created" node), which *baked* the cut into the visible mesh instead of reverting it. Fix is in the shared preview contract: a new `PRESERVE_GEOMETRY = True` opt-in (set on `CutOnAxisSlots`) makes `CleanupContract` snapshot the operated meshes and, on rollback, restore in place any that survived but diverged from the snapshot ‚Äî preserving the original shape's identity/UUID and skipping the restore when node-diff already reverted (so meshes *with* legitimate history keep it). Regression coverage in `TestCutOnAxisPreviewRollback` (`test_edit_tools_geometry.py`) drives the real `Preview` through enable‚Üírefresh‚Üídisable on a historyless cube. Mirror / duplicate (`MUTATES_SELECTION`) paths unchanged.

- **Channels wheel-scroll ‚Äî symmetric ladder + axis-swap fix** ‚Äî wheel-scroll on a channel-box value cell now mirrors uitk `SpinBox`'s symmetric ladder. The two user-visible bugs both root in the table layer:
  - **Alt-held wheels did nothing**: `TableWidget._emit_wheel_scrub` read only `event.angleDelta().y()`, which is 0 on some Qt builds when Alt is held (Qt transposes the delta onto `.x()`). The widget now reads `delta.y() or delta.x()`, so Alt and Ctrl+Alt both fire reliably. (Fix in uitk; the channels module is just the consumer.)
  - **Ctrl+Shift fell to the default 0.1 step**: the old `_wheel_step` matched modifiers with `(mods & MASK) == MASK`, which can return False under PySide6 when the live `KeyboardModifiers` flag set is compared to a `KeyboardModifier` flag value. Detection now uses `bool(mods & SPECIFIC)` per modifier, making the comparison robust to PySide5/6 type quirks.
  - **Modifiers source**: `_on_wheel_scrolled` now takes the modifiers from the `cellWheelScrolled` signal payload (added in uitk) rather than polling `QApplication.keyboardModifiers()`. Removes a race against the wheel event.
  - **Ladder** (matches uitk `WheelStepMixin`):

        plain         √ó1       (default)
        Ctrl          √ó10      (coarse)
        Ctrl+Shift    √ó100     (very coarse)
        Alt           √∑10      (fine)
        Ctrl+Alt      smallest representable step (10‚Åª·µà·µâ·∂ú‚Å±·µê·µÉÀ°À¢; for ints = 1)

    For int attrs `Alt` collapses to 0 (no sub-1 step exists) -- the wheel notch is honestly a no-op there. The earlier *Ctrl+Alt = snap-to-integer* iteration was reverted: it visually matched `Ctrl` for any integer-aligned starting value, so it looked broken even when working correctly. The smallest-step gesture is now distinct from `Ctrl` regardless of value alignment.
