- **HDR Manager — loading an incomplete/truncated HDR no longer crashes Maya (2026-06-16).** Root cause (from two `MayaCrashLog` dumps): an `ACCESS_VIOLATION` in `OGSATILIntegration.dll!AtilImageHandler::GetIBLIntensity` ← `LoadTexture` ← `AcquireTextureFromFileNode` ← `OGSApplicationBridge::finishLightsTranslation`, on the next viewport idle refresh. Viewport 2.0 loads the skydome's file texture to compute the light's IBL intensity; when the image is **incomplete** the ATIL load returns null and the un-null-checked `AtilImage::Width` dereferences it → hard crash. The offending file was a **partially-synced Dropbox 8K `.hdr`**: its header declared 8192×4096 but only **82 of 4096 scanlines** were actually on disk (1.3 MB). This fired regardless of how the HDR was applied (it's an async viewport refresh), which is why the earlier `cmb000` timing change didn't help. Fix: `HdrManager.hdr_env` setter now **refuses** an existing-but-incomplete image (logs a clear warning, builds nothing) via the new `ptk.ImgUtils.validate_image_integrity` — a hard guard protecting every caller (slots, `create_network`, external code); a merely *missing* file is still allowed (Maya shows a checker, never reaching the IBL path). Slots add UX: `_validate_or_warn` gates the **Set HDR** button, **Add HDR** browse (incl. the `_import_into_sourceimages` "already inside sourceimages — even a subfolder" no-copy path that wired the original file in place), and `cmb000` live-swap, footering "looks incomplete or corrupt (…); re-download or re-export it" instead of crashing. - **Curtain — real "Mid Folds" V-forks, renamed creases, dropped dead seed/presets.** The old **Mid Folds** slider only added V-shaped creases — it never *folded* anything — so it's renamed **Creases** (`creases` + `crease_seed`, was `mid_folds`/`fold_seed`; behavior unchanged). **Mid Folds** is now a genuine adjustment matching how a heavy gathered drape actually forks (`mid_folds` intensity + `mid_fold_seed`): on a seeded random ~1/4–1/2 of the *interior* hang points, a downward **V** apexes *on* the hang point and fans its two arms out and down into the neighbouring spans, interrupting their plain in/out belly.
