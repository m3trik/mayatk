# mayatk — Changelog

## 2026

- **2026-06-07 Cylinder UV unwrap — sews pre-existing UV cuts shut before seaming (default).** `cut_cylinder_seams` / `unwrap_cylinder` cut the auto-seam straight onto whatever UVs the mesh already had, so stray borders from an earlier projection / manual edit survived into the result (e.g. a mesh carrying a 6-shell `polyAutoProjection` came out with 6+ shells instead of the intended 3). They now `polyMapSew` all edges first, so the result's shells come **only** from this operation's seams. New `sew=True` parameter on `_seam_cut_one` / `cut_cylinder_seams` / `unwrap_cylinder` (pass `sew=False` to cut onto the existing UVs). Covered by `test/test_uv_utils.py::TestUvCylinderUnwrap.test_sew_clears_preexisting_uv_cuts` (a cylinder pre-seeded with a messy auto-projection unwraps to 3 clean shells with the default, but keeps the stray shells with `sew=False`). 28/28 uv_utils green.
- **2026-06-07 Cylinder UV unwrap — one bad mesh no longer aborts the whole selection.** `unwrap_cylinder(unfold=True)` unfolded **all** meshes in a single batched `u3dUnfold` call inside one try/except, so a mesh u3dUnfold rejects (`// Error: Mesh has non-manifold UVs. Clean up the mesh before using unfold.`) skipped the unfold for *every* selected mesh. Now each mesh is seeded → unfolded → packed → unflipped **independently**, wrapped per-mesh: a rejected mesh logs `unwrap_cylinder: unfold skipped for <mesh> (<reason>)` and is left seam-cut with its seed projection, while the rest unfold normally. (Per-mesh u3dUnfold is the same result as the batch — shells are independent — and the layout was already per-mesh.) Covered by `test/test_uv_utils.py::TestUvCylinderUnwrap.test_multi_mesh_skips_non_manifold_keeps_good` (two good cylinders flanking a welded-cube non-manifold mesh: no exception, both cylinders unfold to clean shells). 27/27 uv_utils green.
- **2026-06-07 Cylinder UV unwrap — ~6-11x faster pack.** `unwrap_cylinder(unfold=True)` spent ~90% of its time in `u3dLayout`, whose cost is roughly quadratic in its `resolution` (~1.2 s at 4096, ~4.7 s at 8192) — and it was being passed the full `map_size`. But the pack is **pixel-identical from ~256 up** (verified: same shells / 0 degenerate / 0 flipped / same fill on the reference scene) because `shellSpacing` is already normalized, so `resolution` only sets pack precision, not the gap. Capped it at `min(map_size, 1024)`: a turned column now unwraps in ~0.4 s instead of ~2.5 s (map 4096) / ~5 s (map 8192). `u3dUnfold`'s `mapsize` has no such cost and is left at `map_size`. No behavior change (26/26 uv_utils green).
- **2026-06-07 Cylinder UV unwrap — made `unwrap_cylinder(unfold=True)` clean on low-poly tubes and stopped the layout collapsing / mirroring shells.** Testing the unfold path against a second reference mesh (an 8-sided turned vase, single-row bands) surfaced four real defects the earlier 12-sided validation hid; all fixed and the result now matches the hand-authored target exactly (9 shells, 0 degenerate, 0 mirrored, packed in 0-1). (1) **Faceting read as creases** — on a low-poly tube the polygon facet dihedral can meet the crease threshold (an 8-sided tube facets at exactly 45°), so `get_auto_seam_edges` cut every vertical facet edge and shattered the tube. It now classifies **axial-ness first**: an edge running lengthwise (parallel to the revolution axis) is faceting and feeds the single lengthwise column; only a **circumferential (ring) edge** with a sharp dihedral is a genuine step / cap crease. (2) **Band collapse** — the planar `_seed_shell_uvs` folded a tubular band's front and back onto each other (a single-row ring degenerated entirely), which `u3dUnfold` then collapsed. The seed is now **axis-aware**: a band that wraps the axis (radial normals) gets a **cylindrical** projection about the axis (an unrolled, non-folded strip); a cap / flat step keeps the thinnest-axis **planar** projection. (3) **Packer collapse** — `polyLayoutUV` collapses these cylindrically-seeded shells to points, so packing switched to **`u3dLayout`** (the proven `tb000` packer) followed by `polyNormalizeUV(normalizeType=1, preserveAspectRatio=True)` to fit the area-scaled pack back into 0-1. (4) **Packer mirroring** — `u3dLayout` mirrors some shells to pack tighter, leaving the texture mirrored; new `UvUtils._unflip_reversed_shells` flips any reversed (negative-winding) shell back about its own UV center, in place (packing and 0-1 fit preserved). Diagnosed by isolating each pipeline stage (faceting → shells; seed → fold; layout → collapse; pack → mirror) on the reference scene. `cut_cylinder_seams` (seam-only) and the `get_cylinder_seam_edges` loop seamer are unchanged. Covered by two new `test/test_uv_utils.py::TestUvCylinderUnwrap` regressions on an 8-sided cylinder — `test_low_poly_cylinder_seam_ignores_faceting` (faceting not cut → body + 2 caps, not shattered) and `test_low_poly_unfold_clean_shells` (non-degenerate, non-mirrored, in 0-1) — plus a robust `pluginInfo`-based Unfold3D skip guard (the old `if not loadPlugin(...)` skipped whenever the plugin was already loaded). 26/26 uv_utils green in Maya 2025.
- **Cylinder UV unwrap — fixed `unwrap_cylinder(unfold=True)` collapsing every shell to a point, and packed the result.** The seam cutting was correct (right shells), but `u3dUnfold` **collapsed each shell to a zero-area point** whenever the source mesh carried a degenerate, axis-aligned planar UV projection (the common case — e.g. a tube planar-projected along its axis, where each lengthwise band projects to a *line*). `u3dUnfold` can't recover from a zero-area seed. Fix: a new `UvUtils._seed_shell_uvs` planar-projects **each UV shell from the world axis of its thinnest 3D extent** (discarding the flat dimension → guaranteed non-zero-area seed) right before the unfold, so every section flattens instead of collapsing. After unfolding, the shells are **packed into the 0-1 square** (`polyLayoutUV layout=2`, `rotateForBestFit` aligning to U/V when `orient`), so the tool now emits a clean, non-overlapping, in-bounds layout (verified against a real reference scene: 9 shells, 0 degenerate, laid out at least as cleanly as the hand-authored target). The UV pipeline stays as construction history (a consistent chain — `u3dUnfold` itself emits a `polyTweakUV` node — that re-evaluates correctly), so the caller's modeling / deformer history is left intact rather than baked away. Diagnosed by isolating the pipeline step-by-step (seam cut → shells correct but bands already zero-height from the projection → `u3dUnfold` → all 9 collapsed). `cut_cylinder_seams` (seam-only, no unfold) is unchanged. Covered by `test/test_uv_utils.py::TestUvCylinderUnwrap.test_unwrap_unfold_does_not_collapse` (deliberately seeds a degenerate `md="y"` projection, then asserts every shell has non-zero UV area and is packed into 0-1). 24/24 uv_utils + 24/24 curve_to_tube green in Maya 2025.
- **Cylinder UV unwrap — rewritten as a single hard-crease + axial-column auto-cut (`get_auto_seam_edges`).** Replaces the earlier `segment_angle` experiment (removed): one clean algorithm now peels a turned / stepped cylinder into per-section UV shells, validated against a real reference scene (a 12-sided turned baluster: produces the same 9 shells / 102 seam edges as the hand-authored target). New `UvUtils.get_auto_seam_edges(mesh, angle=45, invert_seam=False)` returns two complementary cuts: (1) **hard creases** — every edge whose two faces meet at ≥ `angle`° (cap rims + ~90° step rings), so each smooth section / flat step / cap separates while shallow chamfers stay merged with their neighbour; (2) **one lengthwise column** — a single column of axial edges at one angular position about the tube axis, which opens every tubular section into a flat strip (flat steps & caps are already planar, so they keep their shape and need no opening cut). The tube axis is the odd-one-out eigenvector of the vertex covariance (`_revolution_axis`, robust whether taller-than-wide or wider-than-tall); 3D boundary edges (an open tube's rims) are already UV borders and left uncut; the column is chosen by `_pick_axial_column` (adaptive angular window from the inter-column gap, `invert_seam` lands it on the far side). `cut_cylinder_seams` / `unwrap_cylinder` now take `angle` (was `segment_angle`) and route through it; `unwrap_cylinder` is the single "auto unwrap cylinder" entry point. `get_cylinder_seam_edges` (lengthwise-loop + caps) is **reverted to its prior form and retained as the lower-level seamer for Curve to Tube's smooth swept tubes** (curved tubes break the straight-axis assumption, so they keep the loop seamer — its 24 tests stay green). Covered by `test/test_uv_utils.py::TestUvCylinderUnwrap` — `test_auto_seam_smooth_cylinder_three_shells`, `test_auto_seam_stepped_cylinder_five_shells` (2 caps + 2 bands + flat step annulus, Euler stays 2), `test_angle_threshold_controls_creases` (a 120° threshold treats the 90° steps as soft), `test_auto_seam_invert_opposite_column`. 23/23 uv_utils + 24/24 curve_to_tube green in Maya 2025.
- **Bevel — fixed the preview/undo system (lost material, re-beveled different edges, stacked on every value change) and the launch crash.** Two issues, both only visible once the panel actually opened: (1) **Launch crash** — `BevelSlots.__init__` registered the Width/Segments spinboxes as deferred widgets (`connect_multi`) *before* wrapping them in option-box reset buttons (`add_reset_buttons`); the wrap reparents each spinbox and invalidates its Python wrapper, so `_process_deferred_widgets` crashed with `Internal C++ object (QSpinBox) already deleted` and the marking-menu panel never opened. `add_reset_buttons` now runs first (and the same reorder was rolled out to every sibling preview tool — Mirror, Cut On Axis, Bridge, Duplicate Linear/Radial, Curtain, Curve to Tube — that had the latent ordering; the underlying switchboard was *also* hardened to self-heal a reparented deferred widget — see uitk CHANGELOG). (2) **Broken rollback** — `BevelSlots` didn't declare `PRESERVE_GEOMETRY`, so `polyBevel3` (in-place, with construction history) on a **historyless** mesh (frozen/imported/combined) had its auto-created orig-shape deleted by the preview's node-diff rollback, **baking the bevel in**: each value change stacked another bevel, the baked topology renumbered the edges so the captured `e[0]` then pointed at a *different* physical edge, and the shape swap dropped the material. `BevelSlots.PRESERVE_GEOMETRY = True` (mirrors Bridge / Cut On Axis) makes Preview snapshot the owning mesh and restore it (topology + edge numbering + material) on rollback, so every refresh re-bevels the same captured edges from a pristine mesh. Covered by `test/test_edit_tools_geometry.py::TestBevelPreviewRollback` (declares the opt-in; refresh on a historyless cube matches a clean single bevel and disable restores counts, `e[0]` position, and material — the position check catches the renumbered-edge symptom that symmetric face counts miss).
- **Preview — fixed multi-material meshes losing their shading through the hermetic preview (the whole object rendered bright green).** Single-material (object-level) shading survived the geometry snapshot/restore, but **per-face / multi-material** assignments did not: the rollback's node-diff deletion of the op's `groupId` nodes (and the `delete(ch=True)` in the in-place geometry restore) strip the component shading. **Investigated whether this could be fixed "at the source" by having the rollback's in-place geometry restore also copy the per-face shading from the duplicate `PRESERVE_GEOMETRY` already holds — it cannot.** Empirically (multi-cycle real-Preview repros under `maya.standalone`): reasserting per-face shading onto a *bare in-place-rebuilt* mesh (right after `outMesh→inMesh` + `delete(ch=True)`) leaves the shape in BOTH a whole-object set and component sets at once, an overlap Maya silently tolerates until the **next poly op collapses it to the whole-object material and drops every other one**. Per-face shading only sticks when reasserted on a **freshly-built** mesh. So `Preview` snapshots each operated mesh's shading **as data, once, at enable** (`MatUtils.get_shading_assignments` → `{shading_group: None | [face_indices]}`, keyed by transform UUID) and reasserts it (`apply_shading_assignments`) **after each clean op** — after the forward preview build, after the **commit** replay (inside the undo chunk), and after the **disable** rollback. `apply_shading_assignments` was hardened to drive multi-material assignments **entirely through face components** (a clean-slate `initialShadingGroup` pass, then each material as components, then the dominant material base-coats any faces the snapshot doesn't cover — e.g. a bevel's new chamfer faces): a whole-object base coat is exactly what created the whole+component overlap above, so it's avoided. Single-material snapshots are still applied as one clean whole-object assignment. This lives in the shared `core_utils/preview.py`, so it fixes Bevel, Bridge, and Cut On Axis alike. New `MatUtils.get_shading_assignments` / `apply_shading_assignments` primitives in `mat_utils`. The snapshot is only taken for meshes that actually carry per-face material (object-level shading survives the preview natively), so single-material tools pay nothing. Covered by `TestBevelPreviewRollback::test_preview_preserves_multi_material_through_commit` (matA whole-object + matB on one face survive enable→refresh→commit with zero green faces, asserted live after enable and after the value-change refresh too) and `TestMatUtils` round-trip / base-coat cases; geometry 38/38, mat_utils 35/35, preview 28/28, bridge 9/9, curtain 61/61, curve_to_tube 24/24, duplicate 45/45 green under `maya.standalone`.
- **Duplicate Grid — fixed instance mode mutating the source's shape (switching Instance↔Unique in preview deleted the object, leaving a husk of empty transforms).** Instance mode built the prototype by instancing the *source itself* (`cmds.duplicate(obj, instanceLeaf=True)`), so the copies shared the source's own shape; reparenting those instances during replication then made Maya **renumber that shared shape** (`pCubeShape` → `pCubeShape4`). That both mutated the user's node (violating `duplicate_grid`'s documented no-mutation contract) and corrupted the hermetic preview's path-based rollback: the renamed source-shape path landed in the contract's `created` set, defeating the `_shares_surviving_instance` guard, so a mid-preview refresh (e.g. the combobox switching modes) could delete the shared shape — wiping the source and leaving the grid as empty transforms. Fix: the prototype is now **always a real copy** (never `instanceLeaf`); instance mode shares one **fresh** shape across the *copies* via the instanced replication, leaving the source pristine (`mode="instance"` copies still share a single shape — just not the source's). Regression test `test_instance_mode_leaves_source_shape_pristine` (source shape same node/name, copies share a shape that is not the source's) fails on the old build and passes now; full grid suite 25/25 green under `maya.standalone`. (`DuplicateGridSlots.MUTATES_SELECTION` stays as a cheap defensive backstop — the source is no longer touched, so rollback's node-diff already restores cleanly, but it guards the headlessly-untestable interactive path.)

- **Duplicate Grid — fixed the preview deleting the user's source object, rewrote the build for speed, and gated bulk grids behind a confirmation.** Three issues:
  - **Source destroyed on preview rollback (both instance and non-instance modes).** `duplicate_grid` grouped the originals into a temp node and flattened the copies *through world*, which vacated the originals' short names and let a copy collide with them ("More than one object matches name"). Preview's path-based diff then couldn't tell them apart and deleted the source — its UUID was gone after toggling Preview off (verified in Maya). The rewrite **never reparents or renames the originals** — it reads their bounding box in place and only duplicates them — so there's no name collision and the source is untouched (matching how `DuplicateLinear`/`DuplicateRadial` already work). `DuplicateGridSlots` also now declares `MUTATES_SELECTION = True` for the same instance-shared-shape UUID-restore backstop as linear/radial.
  - **Slow builds.** The old path built a deep nested hierarchy and then **flattened it one cell at a time** (an `O(X·Y·Z)` recursion of per-cell `cmds.parent` + `cmds.ls`). The rewrite replicates *flat* along each axis via a new `DuplicateGrid._replicate_axis` (per axis: duplicate the running group once per step, offset, batch-reparent its children up) for `O(X+Y+Z)` command calls and no post-build flatten, and batches the isolation-set add into one call. Typical grids are now sub-second (10×10×10 ≈ 1s; 10×10×5 ≈ 0.3s). Profiling confirmed this is the best available approach — a per-cell `cmds.instance` loop is ~40× slower (parenting into a growing group is quadratic), and nesting-then-single-flatten is worse still (deep instance hierarchies are slow to duplicate/delete); the residual superlinear cost at very large N is intrinsic to Maya's instance duplication.
  - **Accidental huge grids.** `DuplicateGridSlots._confirm_bulk` prompts (`sb.message_box` Yes/No) before building when the total copy count exceeds `BULK_THRESHOLD` (1000). The approved magnitude is cached so a confirmed build doesn't re-prompt on every preview refresh; declining during a live preview switches the preview off (deferred, since `disable()` no-ops mid-refresh) so dragging the count up doesn't re-prompt each tick.
  - Covered by `test/test_edit_tools_duplicate.py`: `TestDuplicateGridPreviewRollback` (source + its shape survive enable→disable→refresh in both modes; scene left clean) and new `TestDuplicateGrid` cases (no reparent/rename of the source, grid positions, negative-direction layout, `MUTATES_SELECTION` declared).

Each fold arm is a **Ricker/Mexican-hat wavelet** (`_ricker()`, the zero-integral negated 2nd-derivative-of-gaussian used for wrinkle synthesis), so a crease pushes the cloth *out* at its line and *in* to either side — material-conserving the way fabric buckles, rather than ballooning one way (fixes folds reading only as outward bulges); arm spread + fold width are span-relative so the fork shape is invariant to hang-point count, while depth stays absolute. The dead **Seed** field (`seed`) is gone — it was never wired into the engine and drove only the always-subtle grain/jitter, which now uses a fixed internal seed (`_BASE_SEED`). **`irregularity`** is now **coherent, band-limited** surface grain — a few smooth zero-mean wave octaves (`_make_billow`/`_billow_offset`, 1/f amplitude roll-off, horizontal frequency leading vertical so it reads as soft vertical-ish wrinkles) — replacing the old per-vertex white noise that read as static; the per-vertex RNG is dropped from `_drape`. UI: `grp_folds` reordered to Mid Folds (`s019`, new) / Mid Fold Seed (`s010`, repurposed from the dead Seed) / Creases (`s014`) / Crease Seed (`s015`) / Irregularity (`s008`). - **New tool — Curtain (`edit_utils/curtain.py`).** Procedural draped-cloth generator, split by responsibility (SRP): **`Rail`** (rail geometry — generate / resolve-from-selection / sample / measure / resample), **`CurtainMesh`** (the *deformation* — drape a grid into pleated, gravity-sagged cloth), and **`CurtainRig`** (the *rig* — `_add_wire` deformer + `_add_clusters` controls, orchestrated by `attach`). The cloth hangs from a *rail* — a selected NURBS curve, polygon edge path, locator chain, or a generated **straight** rail (bow it with `curvature`; the default is flat) — and is pinned at N evenly-spaced **hanging points**. Each hanging point is a pleat (the fabric gathers there); the spans between them belly into folds and sag down a true **catenary** (`y = a·cosh(x/a)`, `catenary_shape()`) — wider gaps fall further, scaled by `gravity`. Controls: `height`, `hanging_points`, `gravity`, `tension` (catenary shape; peak stays at 1 so `gravity` owns the depth), **`round_points`** (round the sharp cusp at each pin into a smooth dome — `sag_profile()` blends the catenary toward `sin²`), `fullness`, `taper` (top-gather ↔ hem-flare), **`mid_folds`** + **`fold_seed`** (seeded **V-shaped creases** that radiate down from random points near the top and run various lengths — the diagonal break-lines of gathered fabric), **`end_bend_left`/`end_bend_right`/`end_bend_falloff`** (signed per-end sideways bend with an edge ramp — e.g. panels curling toward the camera), `irregularity` (now dialled much subtler — the deliberate relief comes from fullness/mid_folds), `seed`, `density` (segments/unit), `reduce` (polyReduce decimation %), `thickness`, **`invert`** (reverse normals), `closed`.
