# mayatk — Changelog

## 2026

- **Curtain — every parameter field gained a *disable* (reset-to-default) button (uitk `DisableOption`).** Each spinbox now carries a small option-box icon button that bypasses the field to its default (greyed out) and restores your value on the next click — a one-click way to neutralise a single parameter. Non-persistent (each session starts with all parameters active). Wired via `CurtainSlots._add_param_disable_buttons` looping `sb.option_box.set_disable()` over the Hanging/Drape/Folds/etc. spinboxes (the X/Y/Z **Position** triplet is skipped — it shares a tight row with **Get**); the default is resolved from the panel's `StateManager` at click time, so no per-field wiring. The reusable plugin lives in uitk (see its CHANGELOG). Because the option-box wrapping reparents each spinbox into its own container, **`_align_spinbox_prefixes` now buckets by the nearest `CollapsableGroup` ancestor** (not the immediate parent) so the per-section value-column alignment survives the wrapping.
- **Curtain — random hanging-point *spacing* (`hang_jitter` + `hang_seed`, Hanging group `s023`/`s024`).** The hanging points were always evenly spaced; `hang_jitter` (0–1) now randomizes the *interior* spacing (the outer ends — and a closed rail's seam — stay pinned so the cloth still spans the full rail), with `hang_seed` picking the pattern. Spans become uneven, and a wider gap bellies and **sags further** (the catenary now scales by the span's own width, not `total/spans`). The drape's span math was refactored behind `CurtainMesh._make_hang_points` (the u-positions, computed in `__init__` so the offset helpers always have them) + `_span_at(u)` (maps a rail position to its `(span, local t)` via `bisect`); `_drape`/`_sway_offset` use it and mid-fold forks anchor on each hang point's *actual* (jittered) position. Jitter caps each interior shift at 0.4 of a span, so points stay strictly ordered (no zero-width spans). **Identical to before at `hang_jitter=0`** (`_span_at` reproduces the old `u*spans` phase — covered by a test). New `test_curtain.py` cases: off-is-uniform, interior-perturbed/ends-pinned/strictly-increasing, seed-changes-pattern, drape-shifts, and phase-parity.
- **Curtain — Set Position for the generated rail + a *Get* button that grabs the selection's bbox center (Rail group, `s025`/`s026`/`s027` + `b002`).** The generated rail was always centered at the origin; a Position (X/Y/Z) now sets where it's centered, and a small **Get** button to the right of the fields fills it from the **bounding-box center of the selected object(s)** (combined world bbox via the `BoundingBox` helper; ignores the panel's own auto-rail; sets all three fields with signals blocked then fires one re-drape). `Rail.make`'s `y` param was generalized to `center=(x, y, z)` (no external callers used `y`); `_field_rail` passes the Position through and `_sync_driver`'s rebuild signature now includes it so moving the rail re-syncs the driver curve. Covered by `MakeRailTest` (center offsets the straight rail's midpoint / the closed ring's center, keeps it level at center-y).
- **Select Result is now a first-class Preview feature (like the Preview/Create buttons), not per-panel boilerplate.** `core_utils/preview.Preview` gained two optional constructor args — `select_result_checkbox` and `result_provider` (a callable returning the operation's current result node name(s)). When both are passed, Preview owns the whole feature itself: it (de)selects the result after **every** preview build and on **commit** (deferred), per the toggle, and wires the checkbox live — so any panel on the Preview system gets Select Result by passing two kwargs instead of hand-wiring it. On commit the apply runs **after** `finalize_func` (so cleanup that changes the selection — e.g. Curtain discarding its auto-rail — can't clobber the result); during preview it runs right after the operation, outside the contract. Default off → fully backward-compatible. **Curve to Tube and Curtain migrated** to it: each now passes `select_result_checkbox` + `result_provider=lambda: self.last_*` and **drops** its `_apply_result_selection` method, the `perform_operation`/`_finalize` apply-calls, and the `chk00N.toggled` wiring (Curtain's `_finalize` is now just the auto-rail discard). The selection mechanics still live in the shared `apply_result_selection` primitive (`object_mode=True` always, since the result is selected as an object and any leftover component selection is collapsed first). Coverage moved to the right layer: new `test/test_preview.py::TestPreviewSelectResult` (on/off, live toggle, component-selection clear, unconfigured no-op, provider-error safety, full enable→commit lifecycle, plus primitive missing-widget fallback + deferred-reread-settles); the panels' duplicated selection tests were removed (preview 28 / curtain 51 / curve_to_tube 24, all green).
- **Select Result — fixed a preview started with the toggle *off* still showing the result selected (shared `apply_result_selection`).** Same class of bug as Curve to Tube: an op can leave its result in a **component** selection (the Curtain build ends with `polySoftEdge`, which leaves the mesh's edges selected; Curve to Tube's seam cut leaves map components) — and a transform-only `select(deselect=True)` can't clear that, nor does `selectMode(object=True)` collapse an existing component selection (verified in mayapy). The shared `core_utils/preview.apply_result_selection` now, under `object_mode`, **collapses any leftover component selection onto the result objects (`select(replace=True)`) before deselecting**, so "off" reliably clears it. `CurtainSlots._apply_result_selection` now passes `object_mode=True` (it previously omitted it — that's why the curtain preview-initial-off still selected). Covered by `test/test_curtain.py::SelectResultTest::test_off_clears_leftover_component_selection` (seed a `.e[0:5]` selection, toggle off → nothing selected); Curve to Tube's 28 selection tests stay green (the collapse is a no-op there since its `create` already pre-collapses to an object selection).
- **Curtain — mid-folds run nearer the top + wider/varied widths; Sway cap raised 3 → 6.** The mid-fold forks previously faded in over a wide top fade (`_MIDFOLD_FADE` 0.12), leaving the top ~12 % of the drop fold-free; the top and tip fades are now **decoupled** — a small `_MIDFOLD_TOP_FADE` (0.04) ramps the fork in just under the rail so it reads nearly to the hooks, while the wider tip fade still peters out the lower end smoothly. The **rail row itself (v = 1, the hook line) stays pinned** (the small fade is still zero exactly at the rail). Fork **widths widened and vary more** (`width_frac` ranges 0.18–0.42 / 0.45–0.85, was 0.18–0.30 / 0.35–0.55) so forks affect a bit more area with more variety; selection/anchoring/randomization unchanged. The **Sway** slider max is raised from 3 to 6 (`s020`; `CurtainMesh.sway` was already unclamped above). Covered by `test_midfolds_run_to_near_the_top` (longest fork is near-full strength just below the rail) alongside the still-green `test_midfolds_pin_top_edge` (rail row untouched) and the relief/anchor/push-both-ways tests.
- **Curtain — added a push-pull *Gather* control at the hanging points (`round_gather`, Drape group `s022`, beside Round).** `round_points` only ever *eases* the cusp into a smooth dome — it reduces sag near each pin but can't rise above the rail or dip back, so it pushes without pulling. Gather adds the missing pull: the fabric puckers **up** above the rail right at each hanging point (the *push*) and **dips** lower just inside it as the slack falls off (the *pull*), easing back out by mid-span so the center sag is untouched — the characteristic ripple of a gathered/pleated header. It's a separate dial (default off, independent of Round), so existing presets/curtains are unchanged. The shape lives in the **pythontk primitive** (`MathUtils.catenary_sag` gained a `gather` arg — the `ring²·(2.4 − 3.4·ring)` lobe, `ring = cos²(πs)`: −1 at the support, +0.18 dip just inside, 0 at center); `CurtainMesh` gained the `round_gather` param and `_drape` threads it through `sag_profile` (calibrated ×0.5 so a full slider lifts the pin ~half a sag-unit, matching the other shaping dials). Covered by `test/test_curtain.py` (`test_gather_puckers_up_at_pins_and_dips_inside` — vs a no-gather build, lifts up at the pins, dips inside, leaves in/out x·z untouched) and pythontk `test_math.py` (push-above-baseline / pull-inside / center-untouched / off-matches-catenary).
- **Curtain — added a lateral *Sway* fold control (`sway` + `sway_seed`, Folds group `s020`/`s021`).** The existing folds only push the cloth in/out (along the in-plane normal); Sway additionally leans a **random subset** of the folds **left or right along the rail** (the in-plane *tangent*), with each fold's direction and amount random per seed — so pushed-in and -out areas drift sideways, not just in/out. The lean rides the belly envelope (`|sin(π·phase)|`, zero at the pinned hang points so adjacent oppositely-leaning spans can't tear, peak mid-span) and grows toward the free hem. `CurtainMesh` gained `sway`/`sway_seed` params, `_make_sway` (per-span signed leans, ~half non-zero) and `_sway_offset`; `_drape` now threads the rail tangent and adds `tan * lateral`. Old presets are unaffected (Sway defaults off). Covered by `test/test_curtain.py` (off→empty, random-subset structure, seed-changes-pattern, pinned-at-hang-points, and a straight-rail build that shifts vertices along X but not in/out Z).
- **Curtain panel — added a Select Result toggle (`chk005`, under the preset combo).** Mirrors Curve to Tube: with it on the curtain is selected on commit (and live during preview) so you can see the result; with it off it's explicitly **deselected**. `_finalize` no longer unconditionally `cmds.select`s the curtain — selection is re-asserted (after the auto-rail discard) per the toggle. The select/deselect logic was promoted to a **shared `core_utils/preview.apply_result_selection(widget, results, *, object_mode, defer)` primitive** (re-reads the checkbox each pass so a settled toggle wins; `object_mode` for ops that leave component selection; immediate + optional idle re-assert); both `CurtainSlots` and `CurveToTubeSlots._apply_result_selection` now delegate to it (DRY — was duplicated). The panel's previously-unused **footer now reports the result's triangle count** (`_update_footer`, updated each build; default hint until then), like Curve to Tube. Covered by `SelectResultTest` + `FooterTest` in `test/test_curtain.py` (on selects / off deselects / missing-widget fallback / live toggle / tri-count readout).
- **Curve to Tube — fixed live-polygon normals flipping when the control curve is edited (curveWarp deformer drive).** Root cause (localized via `test/temp_tests` repros): `cmds.extrude`'s swept-surface normal orientation is **not invariant under path edits** — a large enough control-CV move globally flips the NURBS surface normal (confirmed: surface outward-fraction 1.0 → 0.0), which `nurbsToPoly` and the build-time `_conform_poly_outward` (`polyNormal` reverse) faithfully propagate, inverting the live mesh. No `cmds.extrude` option fixes it (swept all 16 combos of `useProfileNormal`/`fixedPath`/`reverseSurfaceIfPathReversed`/`useComponentPivot` — every one flips) and no baked `polyNormal` mode fixes it (any baked reverse-or-not is frozen at build time while the input orientation isn't). **Fix:** open polygon tubes now build a **straight, already-conformed base mesh** (same `extrude → nurbsToPoly` topology — exact sides, RDP rings, caps, UV seams — laid out along +X with rings at the RDP points' cumulative-chord positions scaled to the driver's arc length) and drive it with a **`curveWarp` deformer** keyed to the (resampled) source curve. A deformer only *repositions* vertices, so the face winding — and the normals — can never flip on a live edit; the warped cross-section is also rounder than the old extrude (constant radius vs the extrude's pinch/bulge on bends). Editing the curve re-warps; baked builds collapse the deformer into the mesh. Live and baked still share topology, and a deformer leaves UVs untouched, so the seams/shells are unaffected. Closed (torus) paths and the no-plugin fallback keep the curved `extrude → nurbsToPoly` build; finishing steps are shared via the new `_finish_poly_tube`. Adds a `curveWarp` plugin dependency (ships with Maya; loaded on demand by `_load_curvewarp`, with graceful fallback). New `_polygon_tube_warped`; `test_live_polygon_normals_stable_on_curve_edit` drives four aggressive edits (incl. the `cv[0]` +5y that used to invert to ~2.5% outward) and asserts the capped tube's signed volume stays positive (outward); `test_live_polygon_source_curve_resampled_and_drives` now asserts a `curveWarp` node in the live graph.
- **Cylinder/tube UV unwrap spun off into reusable `UvUtils` helpers (`uv_utils/_uv_utils.py`).** The seam logic that was private to Curve to Tube now lives on `UvUtils` as a shared, testable surface: `get_cylinder_seam_edges(mesh, sections=None, invert_seam=False, cap_faces=None)` returns `(length_loop, cap_rings)` — one lengthwise edge loop plus the ring(s) where each end cap meets the body — handling **open tube** (boundary → rim-vertex seed), **capped cylinder** (caps as explicit indices *or* auto-detected n-gons, so it's exact for any section count when the caller supplies them), and **closed torus** (the loop whose edge count differs from `sections`). `invert_seam` seeds the loop from the diametrically-opposite start-ring vertex so the seam lands on the back side. `cut_cylinder_seams(objects, invert_seam, history)` cuts those seams (`polyMapCut`); `unwrap_cylinder(objects, invert_seam, unfold, orient, map_size)` cuts then unfolds (Unfold3D) + orients to lay the body out as a clean strip and each cap as its own shell. Covered by `test/test_uv_utils.py` (`TestUvCylinderUnwrap` — capped → 3 shells/topology-preserving, open → 1 strip, invert → opposite-side disjoint loop, seam-edge counts).
- **Curve to Tube polish (`nurbs_utils/curve_to_tube.py`).** UV seaming now **delegates to `UvUtils.get_cylinder_seam_edges`** (caps are closed first, capturing the new cap faces by index so seams are exact for any section count — no more capturing rims before `polyCloseBorder`). Smoothing is now **by angle**: the body is softened (`polySoftEdge 180`) then the **cap rings are re-hardened** (`polySoftEdge 0`) so the tube shades smooth while the cap edges read crisp. The **open ends are now pinned across Path Res**: RDP keeps a closer first-interior point at finer tolerances, swinging the endpoint tangent and visibly rotating/tilting the end rings as Path Res changed; a fixed end-tangent anchor (`_END_ANCHOR_FRAC`, one sample ~3 % in from each open end) holds it, dropping end-ring drift from ~11 %-of-radius to ~0 (only the interior re-tessellates now). The `CurveToTubeSlots` panel gained a **Select Result** toggle (`chk004`, beside Keep History): on it selects the finished tube(s) so you can see the tessellation, **off explicitly deselects** them (the build can leave the new mesh selected, so "off" actively clears it). The (de)select is applied as the final action of **every build — preview AND commit** (`perform_operation`, deferring via `evalDeferred` only when `contract is None`, the Preview's sanctioned commit hook), so the toggle governs the live preview selection too, not only the committed result; on commit it is also the operation's final action (not a separate `finalize_func`), immune to the operation's own leftover selection. **The preview previously ignored the toggle entirely** — `create` always selects its result (to clear the seam components) and the (de)select was only applied on the commit replay, so previewing with Select Result off still selected; applying on every build fixes it (Curve to Tube installs no selection scriptJob, so a preview-time (de)select can't trigger a rebuild). The `chk004` read is defensive (falls back to selecting if the widget is absent) and is **re-read inside the apply on every invocation** (not snapshotted once): on the commit path the `lowestPriority` deferred re-assert then runs *after* any deferred state restore settles and re-reads the correct value, so the settled choice wins. The panel's **footer now doubles as a live stats readout** (`_update_footer`, called after each build): polygon → `tris · faces · verts`, NURBS → span counts, multi-curve prefixed with the tube count; it updates as the preview re-sweeps and shows a hint when idle. Covered by `test_update_footer_reports_stats`. **Caps now defaults OFF**. Covered by new `test_polygon_caps_have_hard_edges` (cap-ring edges hard, body soft) and `test_polygon_ends_fixed_across_path_res` (end-ring vertices invariant across Path Res), plus the existing UV-seam/topology tests (straight-minimal counts bumped +2 for the end anchors).
- **New tool — Curve to Tube (`nurbs_utils/curve_to_tube.py`).** Sweeps a circular profile along selected NURBS curve(s) into a tube, output as a **NURBS surface** or a **polygon mesh** (the `CurveToTubeSlots` panel's `cmb000` picks the type and greys out the options that don't apply). `CurveToTube.create(curves, output_type, radius, sections, path_divisions, degree, caps, quads, live, cleanup)` is pure geometry (classmethods returning node names), callable straight from a script. **NURBS** path extrudes a `circle` along the curve (`extrudeType=2`, component-pivot + fixed-path + profile-normal); it **self-conforms outward** by reversing the U direction when a mid-path ring's normal points inward (the extrude could emit either orientation depending on how the curve was drawn — it was coming out inverted) and sets a fine **display smoothness** (the viewport `3` preset) so the surface renders smooth, not faceted. **Polygon** path places rings by **RDP curvature simplification**: the curve is densely sampled and run through `pythontk.MathUtils.simplify_rdp` (Ramer-Douglas-Peucker), which keeps points where the path bends and drops near-collinear runs. A working curve is built **through** those kept points (`cmds.curve`, periodic-closed for loops), the profile circle is extruded along it, and `nurbsToPoly(uType=2, uNumber=sections, vType=1)` converts the surface to *exactly* `sections` sides around with one ring per span — so rings land **only** at the curvature-selected points, dense on tight bends, sparse on straight runs, and **decoupled from the source curve's CV spacing** (`path_divisions` / "Path Res" scales the RDP tolerance as a fraction of arc length). Building *through* the points (rather than `insertKnotCurve`, which kept the curve's uniform CV-span baseline and over-tessellated the gentle bends) is the key — on a tight-arc-vs-gentle-arc test it concentrates ~4.5× more rings on the tight bend at a far lower total count. The mesh gets `polySoftEdge(180)` for smooth shading and a normal-conform (`polyNormal`) so it faces outward regardless of curve direction, and is **UV-seamed** for a clean unwrap — one `polyMapCut` along the length plus each cap's rim, so a capped tube is 3 UV shells (body strip + 2 caps; the rims are captured on the open tube before `polyCloseBorder` makes them internal). A **Keep History** toggle keeps the construction chain (collapsed to a baked mesh otherwise). For a live polygon the source curve **is resampled in place** to the RDP points (`cmds.curve -replace`, identity/UUID preserved) so it stays the editable live driver; the hermetic `Preview` restores it on rollback (`CurveToTubeSlots.PRESERVE_GEOMETRY = True`, see the Preview generalization below). The hidden inputs (profile circle, NURBS surface) are parented under the tube. A baked polygon builds a throwaway path through the points and leaves the source curve untouched. Live and baked share the same topology. All paths are `cmds`-based, so a committed tube is a single clean Ctrl+Z; a closed/periodic curve makes a watertight torus. (Earlier iterations tried procedural reshaping, Sweep Mesh, `insertKnotCurve`, and `rebuildCurve rebuildType=4` — none combined exact sides, a clean tube, *and* strong bend concentration the way RDP-through-points does.) Driven by the hermetic `Preview`, validated to require a NURBS curve. Registered `nurbs_utils.curve_to_tube: CurveToTube`; launched from tentacle's NURBS panel (**Curve to Tube** button, `b058`). Covered by `test/test_curve_to_tube.py` (19 tests — exact sides around, RDP bend-focus + **known tight/gentle-arc concentration ratio**, path_divisions monotonicity, identical live/baked topology, outward normals, watertight caps/torus, **UV-seam shells**, source-curve resampled-and-drives, clean single-undo, validation).
- **Preview `PRESERVE_GEOMETRY` generalized from meshes to meshes + NURBS curves (`core_utils/preview.py`).** The hermetic Preview's in-place geometry restore (`CleanupContract`) — which snapshots preserved originals and, on rollback, rebuilds any that survived but were mutated in place — was mesh-only (`_mesh_signature`/`_restore_mesh_in_place` returned/handled nothing for non-mesh shapes). Renamed to `_geo_signature` / `_geo_diverged` / `_restore_geo_in_place` and taught them NURBS curves (signature = spans/degree/form + world bbox; restore = feed the duplicate's `local` into the original's `create`, then `delete -ch`). This is what lets Curve to Tube resample its source curve in place under a live Preview and have it restored cleanly on toggle-off / refresh. Mesh behavior is byte-for-byte unchanged (the mesh branch is the original logic); `test_edit_tools_geometry.py` (CutOnAxis preserve-rollback), `test_preview.py`, and the Mirror/Duplicate preserve paths stay green.
- **`EditUtils.dissolve_coplanar()` — planar decimation (limited dissolve).** Angle-driven, feature-preserving complement to `decimate`: walks each mesh's interior edges (OpenMaya) and `polyDelEdge(cleanVertices=True)`-merges those whose two faces are within `angle_tolerance` degrees of coplanar, leaving creases/corners/silhouette/borders untouched and keeping clean n-gons (no triangulation). At a tiny tolerance it's **lossless** on hard-surface meshes (a 5×5×5 subdivided cube → 6 faces, bounding box unchanged); curved surfaces are left alone (use `decimate`/QEM there). Tentacle's Subdivision **Decimate** button (now `tb000`) exposes both algorithms behind an option-box combobox — Reduce (QEM %) vs Planar (Coplanar) — with the irrelevant options greyed out per selection. Tests: `test_edit_utils.py` (cube→6 lossless, sphere untouched).
- **`EditUtils.decimate()` — reusable polyReduce helper; Curtain math promoted to pythontk.** Extracted the scattered `polyReduce` logic (curtain's mesh reduce + the Subdivision panel's Reduce button) into one boundary-preserving **`EditUtils.decimate(objects, percentage, preserve_borders, preserve_hard_edges, preserve_uv_borders, preserve_quads, symmetry, …)`** (undoable, deletes history by default; early-returns at `percentage <= 0`); the Curtain generator's `reduce` now routes through it, and tentacle's new Subdivision **Decimate** button calls it. Separately, the Curtain module's general math/signal primitives were promoted to `pythontk.MathUtils` (the SSoT): `smoothstep`, `ricker`, `catenary`/`catenary_sag`, `safe_normalize`, vector `lerp`, and the coherent grain generator (now `pythontk.BandLimitedNoise`). `curtain.py` consumes those via thin aliases (`catenary_shape`/`sag_profile` remain importable for back-compat) and keeps only the curtain-specific `_v_arms` fold composition local. No behavior change — the noise is bit-identical (same seed/octave draw order); `test_curtain.py` (43) + `test_edit_utils.py` (+2 decimate tests) stay green.
- **Curtain — real "Mid Folds" V-forks, renamed creases, dropped dead seed/presets.** The old **Mid Folds** slider only added V-shaped creases — it never *folded* anything — so it's renamed **Creases** (`creases` + `crease_seed`, was `mid_folds`/`fold_seed`; behavior unchanged). **Mid Folds** is now a genuine adjustment matching how a heavy gathered drape actually forks (`mid_folds` intensity + `mid_fold_seed`): on a seeded random ~1/4–1/2 of the *interior* hang points, a downward **V** apexes *on* the hang point and fans its two arms out and down into the neighbouring spans, interrupting their plain in/out belly. Lengths vary (some stop high, some run nearly to the hem) as do width/depth (narrow-deep vs wide-shallow); the seed does most of the look's work (`_make_midfolds`/`_midfold_offset`). Modelled in global `u` (continuous across hang points by construction — no per-span cracks) on a shared `_v_arms()` primitive that also backs `_crease_offset`. Each fold arm is a **Ricker/Mexican-hat wavelet** (`_ricker()`, the zero-integral negated 2nd-derivative-of-gaussian used for wrinkle synthesis), so a crease pushes the cloth *out* at its line and *in* to either side — material-conserving the way fabric buckles, rather than ballooning one way (fixes folds reading only as outward bulges); arm spread + fold width are span-relative so the fork shape is invariant to hang-point count, while depth stays absolute. The dead **Seed** field (`seed`) is gone — it was never wired into the engine and drove only the always-subtle grain/jitter, which now uses a fixed internal seed (`_BASE_SEED`). **`irregularity`** is now **coherent, band-limited** surface grain — a few smooth zero-mean wave octaves (`_make_billow`/`_billow_offset`, 1/f amplitude roll-off, horizontal frequency leading vertical so it reads as soft vertical-ish wrinkles) — replacing the old per-vertex white noise that read as static; the per-vertex RNG is dropped from `_drape`. UI: `grp_folds` reordered to Mid Folds (`s019`, new) / Mid Fold Seed (`s010`, repurposed from the dead Seed) / Creases (`s014`) / Crease Seed (`s015`) / Irregularity (`s008`). Dropped the **Flat Backdrop** and **Booth Ring** built-in presets (Stage Swag + Shower Curtain remain, with their creases preserved). Tests renamed (`test_creases_*`) and extended (`test_midfolds_*`, incl. direct checks that forks anchor on a hang-point subset and leave the top edge pinned).
- **New tool — Curtain (`edit_utils/curtain.py`).** Procedural draped-cloth generator, split by responsibility (SRP): **`Rail`** (rail geometry — generate / resolve-from-selection / sample / measure / resample), **`CurtainMesh`** (the *deformation* — drape a grid into pleated, gravity-sagged cloth), and **`CurtainRig`** (the *rig* — `_add_wire` deformer + `_add_clusters` controls, orchestrated by `attach`). The cloth hangs from a *rail* — a selected NURBS curve, polygon edge path, locator chain, or a generated **straight** rail (bow it with `curvature`; the default is flat) — and is pinned at N evenly-spaced **hanging points**. Each hanging point is a pleat (the fabric gathers there); the spans between them belly into folds and sag down a true **catenary** (`y = a·cosh(x/a)`, `catenary_shape()`) — wider gaps fall further, scaled by `gravity`. Controls: `height`, `hanging_points`, `gravity`, `tension` (catenary shape; peak stays at 1 so `gravity` owns the depth), **`round_points`** (round the sharp cusp at each pin into a smooth dome — `sag_profile()` blends the catenary toward `sin²`), `fullness`, `taper` (top-gather ↔ hem-flare), **`mid_folds`** + **`fold_seed`** (seeded **V-shaped creases** that radiate down from random points near the top and run various lengths — the diagonal break-lines of gathered fabric), **`end_bend_left`/`end_bend_right`/`end_bend_falloff`** (signed per-end sideways bend with an edge ramp — e.g. panels curling toward the camera), `irregularity` (now dialled much subtler — the deliberate relief comes from fullness/mid_folds), `seed`, `density` (segments/unit), `reduce` (polyReduce decimation %), `thickness`, **`invert`** (reverse normals), `closed`. Mesh built once via `polyPlane(ch=False)` + `MFnMesh.setPoints`; vector math delegates to `ptk.MathUtils`. `CurtainSlots` drives it through the hermetic `Preview` (live re-drape via `connect_multi`); a rail is **auto-created on preview** when none is selected (so you never have to select an unrelated object) and dropped on commit. In generated mode the drape reads **Width/Curvature live each refresh** (previously the captured auto-rail froze them, so they appeared to do nothing); the owned driver curve resyncs only when a rail-shaping field actually changes. The **preset combo lives in the panel** (`cmb000`); ships built-in presets (Flat Backdrop, Stage Swag, Shower Curtain, and a closed-ring Booth Ring) through uitk's `PresetManager` (built-in tier from `edit_utils/presets/curtain/`). The panel groups its fields into collapsible `uitk.CollapsableGroup` sections (Rail, Hanging, Drape, Folds/Noise, Ends, Mesh — the niche Folds/Noise and Ends start collapsed, each section's open/closed state persisted across sessions) with spinbox value columns aligned per group via font metrics (`_align_spinbox_prefixes`). Mesh-softening is always on (no toggle). `CurtainRig` (wire deformer + per-CV cluster controls) remains a tested library primitive for programmatic rigging but is no longer wired into the panel. Registered `edit_utils.curtain: (CurtainMesh, Rail, CurtainRig)`; launched from tentacle's Deformation panel. Covered by `test/test_curtain.py` (35 tests).
